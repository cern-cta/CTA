* Make sure you have tnsnames.ora in your home directory, wich defines the db connections.

*Install required oracle libraries:
	cx_Oracle
	oracle-instantclient-basic
	oracle-instantclient-devel
	oracle-instantclient-tnsnames.ora

*Install networkx library for python

*Allow server in /etc/sysconfig/iptables (example for port 8000):
	-A RH-Firewall-1-INPUT -p tcp -m tcp --dport 8000 -j ACCEPT
	-A RH-Firewall-1-INPUT -m state --state NEW -m tcp -p tcp --dport 8000 -j ACCEPT
	
*Restart the service to enable changes
	/etc/init.d/iptables restart 

*Install modpython and edit python.conf (example included in the package under /etc/httpd/conf.d/python.conf)

*The Oracle cx libraries doesn't work well with scientific linux by default, you will have to type as root
	setenforce 0

*Start memcache to enable caching by typing
	memcached -u kblaszcz -m 2 -vv

*Copy the content of html directory to /var/www
	set rw rights to /var/www/html/images/treemaps

*Set parameters in settings.py
	DATABASE_PASSWORD = 'thepassword'
	
	CACHE_MIDDLEWARE_KEY_PREFIX = 'some random character sequence'

	#Apache URL that serves the files
	PUBLIC_APACHE_URL = "http://pcitdmssd"
	
	#local URL that contains the files
	LOCAL_APACHE_DICT = "/var/www/html"

	#Where to save the generated images, same file name results in an overwrite
	REL_TREEMAP_DICT = "/images/treemaps"
	
	#Where to find icons
	REL_ICON_DICT = "/images/icons"
	
	TEMPLATE_DIRS = 
	(
    	'/var/www/html/OverView/templates'
	)
	
	DJANGORESPONSE_URL = "http://pcitdmssd"
	
	REL_TREEMAP_DICT = "/images/treemaps"
	
*Start the server by typing 
	service httpd start
