Options inside of the URL are supposed to influence the behavior of treeView and groupView.

1. Presets.py
	Enter all your options for given Preset in Presets.py:
	
	...
	optionsset = [DateOption('start', 'start', 120), DateOption('stop', 'stop', 0), BooleanOption('Lowest values', 'smalltobig', False), BooleanOption('Flat view', 'flatview', False)]
	presetdict["Requests CMS from last 120 minutes"] = Preset(lr, False, 'Requestscms', '/castor', 23, optionsset)
	
	You can write your own Option classes. From outside, they must behave the same as the existing ones.
    
2. Use OptionsReader in views.py
	You can access the options in views.py like that:
	
		from sites.dirs.OptionsReader import OptionsReader
		
		def treeView(request, options, presetid, rootmodel, theid, refresh_cache = False):  
			optr = OptionsReader(options, presetid)
			myoption = optr.getOption('myoption')
			#do something with myoption
		
	The options string may contain errors because the user can potentially manipulate the url. 
	Also, by switching from one view to another some options may not be valid any more and they shouldn't be continued. 
	For example the start and stop options only apply to Requests and not to other presets.
	You also have to make sure that you don't reuse a trashy options string for url generation.
	The best way is to overwrite options by generating them from scrach, based on the values you were able to read from the messy string. 
	That's what getCorrectedOptions() is for. 
	
	If you enter a url like this:
		http://pcitdmssd/treemaps/{flatview=true, smalltobig=truekfgktlstart=12.05.2009_16:00:00blabla}1_Dirs_/castor
	
	OptionsReader receives "flatview=true, smalltobig=truekfgktlstart=12.05.2009_16:00:00blabla".
	OptionsReader will be able to find flatview, smalltobig and start values in it. 
	getCorrectedOptions() will ignore 'blabla' and 'kfgktl' and create clean options which result in clean links like this:
	
	http://pcitdmssd/treemaps/{flatview=true, smalltobig=true}1_Dirs_/castor
	
	Notice, that it also cleans the start-option from the link, because it's not valid for Preset with id=1
	
	If you enter
	http://pcitdmssd/treemaps/{flatview=elephant, smalltobig=true}1_Dirs_/castor
	
	getCorrectedOptions() will return "smalltobig=true" and ignore the "flatview=elephant" part. This results in a clean link like this:
	http://pcitdmssd/treemaps/{smalltobig=true}1_Dirs_/castor
